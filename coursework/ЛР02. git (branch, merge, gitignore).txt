LOOLOOOLL﻿ahahahahahahhЦель: Изучить основные команды системы контроля версий git
Алгоритм действий:
1. Скачать и установить программу git с официального сайта. Дальнейшая работа осуществляется при помощи git bash вызываемое из контекстного меню.
2. Выполните начальную настройку git. Обязательно в параметре user.email указать почту привязанную к github и user.name указать username на github. Остальные параметры могут быть выставлены по желанию.
3. Создать локальную версию репозитория
4. Скачать этот документ в двух форматах: txt, rtf и скачать архив zip. Перенести в корневую папку инициализированного репозитория.
5. Создать в локальном репозитории файл .gitignore и добавить в него игнорирование файлов с расширением zip
6. Определить состояние файлов, добавить на отслеживание и зафиксировать изменения файлов с расширением txt и .gitignore
7. Создать новую ветку с именем test из ветки master
8. Внести изменения в первую строку файла с расширением txt и зафиксировать изменения.
9. Переключиться на ветку master и внести другие изменения в первой строке в файл с расширением txt, и зафиксировать изменения.
10. Выполнить слияние ветки test в ветку master
11. Решить конфликт слияния веток так, чтобы остались изменения из обеих веток. Исключить строки в которых находятся служебные метки конфликта. Зафиксировать изменения файла в котором был конфликт.
12. Написать письмо с прикрепленным заархивированным репозиторием
